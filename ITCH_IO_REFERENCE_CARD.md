## BROWSER BUILD (CRITICAL)

When uploading HTML5 ZIP:
- [ ] Check: "This file will be played in the browser"
- [ ] index.html in ZIP root (NOT in subfolder)
- [ ] ZIP: < 1,000 files, < 500MB total, no file > 200MB
- [ ] Tested in Chrome, Firefox, Safari

**Impact**: 37% engagement rate if done right. 6% if download-only.

---

## WINDOWS BUILD (Optional but Recommended)

When uploading Windows ZIP:
- [ ] Platform: Windows (auto-select)
- [ ] DON'T check "browser playable"
- [ ] Include executable, DLLs, data folder, README
- [ ] Size < 200MB preferred

## BROWSER VS DOWNLOAD DATA

**Browser gamers**:
- 37% of viewers will play (if HTML5 is smooth)
- Lower barrier to entry
- Built-in discoverability

**Download gamers**:
- 6% of viewers will download (much lower friction loss)
- Can play offline
- Preferred by some (performance, mods)

**Recommendation**: Provide both. Browser as primary, Windows as secondary.
