## 2. カテゴリ別診断 + 改善指示

### 2.1 背景 [1/5 → 目標3/5] — CRITICAL

**現状**:
- Stage 1: 単色ダークパープル (#2a1530)
- Stage 2: 単色オレンジブラウン (#8a5a30)
- Stage 3: 単色グリーン (#5a8a30)
- テクスチャなし。パララックスなし。地面タイルなし。環境オブジェクトなし

**VS/Brotato/HoT比較**:
- VS: 草原/墓地/図書館の地面タイル + 2-3層パララックス（遠景の木、中景の柵、近景の草）
- Brotato: 色付きアリーナ床 + 微妙なパターンテクスチャ + 端のビネット
- HoT: 90年代プレレンダ風の詳細な地形（草、道、水辺）

**改善指示（builder）**:
1. **地面タイル追加**: Kenney roguelike-rpg-packの地面タイル（`roguelikeSheet_transparent.png`内の地面セクション）をTileMapレイヤーで敷く。タイルサイズ16px、画面全体+マージン。繰り返しタイル（草/石/土）
2. **2レイヤーパララックス**:
   - Layer -1 (遠景): 暗い装飾物（岩、木のシルエット）をランダム配置。プレイヤー移動の0.3倍速で追従
   - Layer -2 (超遠景): 色付きグラデーション or 星空。プレイヤー移動の0.1倍速
3. **アンビエントパーティクル**: 画面全体にゆっくり浮遊する小粒子（ホコリ/魔法の粒）を30-50個。GPUParticles2D、alpha=0.3、サイズ2-4px
4. **ビネット**: 画面端を暗くするビネットオーバーレイ（ColorRect + shader or テクスチャ）。中央から外に向かってalpha 0→0.4

### 2.2 プレイヤー [2/5 → 目標3/5] — CRITICAL

**現状**:
- 小さな緑三角形（矢印型）or 青い丸
- キャラクター性ゼロ。何のゲームかわからない
- アウトラインなし。背景によっては視認困難

**改善指示（builder）**:
1. **Kenneyスプライトに差し替え**: `kenney/top-down-shooter/PNG/` から適切なキャラクター使用。推奨: `Survivor 1/survivor1_stand.png` or `Man Blue/manBlue_stand.png`。サイズ32x32px以上
2. **1px白アウトライン追加**: ShaderMaterial でエッジ検出アウトライン。`uniform float outline_width = 1.0; uniform vec4 outline_color = vec4(1,1,1,0.8);`
3. **向き表示**: 移動方向にスプライトのflip_hを切り替え。または移動方向に微小回転（±15度）
4. **ヒット時フラッシュ**: ダメージ受けた時にスプライトを白に50ms点滅（ShaderMaterialのuniform）

### 2.3 敵バリエーション [1/5 → 目標3/5] — CRITICAL

**現状**:
- 全敵が「赤いダイヤモンド + 小さなオレンジ矢印」の1種類
- 小型/大型の差はあるが、色とシルエットが同一
- 暗い円形の「影」が付いているが、敵のaura/shadowなのか不明確

**VS/Brotato比較**:
- VS: バット、スケルトン、ゴースト、マミー、ゾンビ等15+種。シルエットだけで区別可能
- Brotato: 各Wave固有の敵デザイン。色+形+サイズで即座に脅威度判別

**改善指示（builder）**:
1. **Kenney roguelikeスプライト使用**: `roguelikeSheet_transparent.png`から敵スプライトを切り出し。最低4種:
   - 小型ザコ: スライム/バット (16x16)
   - 中型: スケルトン/ゴブリン (16x16)
   - 大型: オーガ/デーモン (32x32)
   - ボス: ドラゴン/ゴーレム (48x48 or 64x64)
2. **1px白アウトライン**: プレイヤー同様。背景から浮き出させる
3. **タイプ別カラーリング**: ShaderMaterialでHueシフト。火系=赤、氷系=青、毒系=緑
4. **ヒットフラッシュ**: 被弾時に白フラッシュ50ms（全敵共通、ShaderMaterial）
5. **死亡パーティクル**: 倒した時に破片が4-8方向に散る。CPUParticles2D、寿命0.3s、初速200-400px/s
6. **微小ボブアニメ**: 浮遊する敵は上下に2pxのsin波ボブ（frequency=2Hz）

### 2.4 弾/魔法 [2/5 → 目標3/5] — HIGH

**現状**:
- Fireballは暖色の円形グロー（見える）
- しかしサイズが小さく、トレイルなし、着弾エフェクトなし
- マズルフラッシュなし

**改善指示（builder）**:
1. **弾サイズ最小16px**: 現在の弾が16px未満なら拡大。弾の存在感を上げる
2. **グロートレイル追加**: 弾の後ろに半透明の残像（Line2D or Trail ParticleEmitter）。寿命0.15s、alpha 0.5→0
3. **着弾エフェクト**: 敵に当たった時に小さな爆発パーティクル（8粒子、寿命0.2s、サイズ4-8px）
4. **マズルフラッシュ**: 発射時にプレイヤー位置に白い円形フラッシュ（1-2フレーム、サイズ12px）

### 2.5 HUD/UI [1/5 → 目標3/5] — CRITICAL

**現状**:
- 左上: デバッグテキストそのまま露出（"Move: Kite | Aim: Nearest | Trigger: Auto Cast"、"Bullets: x1"、"Kills: 0"、"Trigger: AUTO: fires on cooldown"）
- スロット表示: "[1] Fireball (1.2s)" "[2] empty" "[3] empty" — 開発者コンソール
- HPバー: 機能するが無装飾の色付き四角形
- XPバー: 灰色の細い四角形
- フォント: システムフォントそのまま

**VS/Brotato比較**:
- VS: ダメージ表示、レベル、HPバーのみ。最小限だが全てスタイル付き
- Brotato: 上部にHPバー（フレーム付き）、レベル表示、武器アイコン。Kenney pixel-ui使用

**改善指示（builder）**:
1. **デバッグテキスト即時非表示**: "Move: Kite | Aim: Nearest..."行を削除 or `#ifdef DEBUG` でリリースビルドから除外。**これが最もチープに見える最大の原因**
2. **スロット表示リデザイン**: テキスト"[1] Fireball (1.2s)"をアイコン表示に変更。画面下部にアイコン3枠。Kenney pixel-ui-packの枠を使う（`9-Slice/Ancient/` or `9-Slice/Colored/`）
3. **HPバーにフレーム追加**: Kenney pixel-ui-packの9-Sliceバー背景を使用。HPバーの上に重ねる
4. **XPバーのスタイル**: 細い灰色→ジュエル色（紫/青系）のグラデーション。フレーム追加
5. **ゲームフォント導入**: Kenney pixel-ui-packに含まれるピクセルフォント or CC0のピクセルフォント（m5x7, Press Start 2P等）。全テキストをゲームフォントに統一
6. **キルカウント/タイマー**: 右上の"9:29"はOK。フォントのみ変更

### 2.6 パーティクル/VFX [2/5 → 目標3/5] — HIGH

**現状**:
- AOEの暖色グロー（大きな円形グラデーション）— これは効果的
- それ以外のパーティクルがほぼ皆無
- 連鎖/コンボ時の視覚的報酬がない

**改善指示（builder）**:
1. **敵死亡パーティクル**: 上記2.3参照
2. **XP吸収エフェクト**: XPジェム回収時にジェム→プレイヤーに向かう光の粒子軌道
3. **レベルアップ爆発**: レベルアップ時に画面全体に放射状パーティクル（48粒子、寿命0.5s）+ 白フラッシュ（1フレーム、alpha=0.3）
4. **ダメージ数字改善**: 現在の平坦なテキスト→ポップアップアニメ（Y方向に-30px移動 + フェードアウト、0.5s）。クリティカルは大きく赤く

### 2.7 テキスト演出 [2/5 → 目標3/5] — HIGH

**現状**:
- "LEVEL UP! Lv.4" — 黄色プレーンテキスト、複数が重なって表示される
- "25 KILLS! BREAKOUT!" — 重なり読めない
- "MASSACRE! x24" — 右上に表示されるが小さい
- "STAGE 2: SURGE!" — 目立つが装飾なし
- "⚠ PRESSURE" — 警告テキスト、機能的だがチープ

**改善指示（builder）**:
1. **テキストキュー制**: イベントテキストは1つずつ表示。前のテキストが消えてから次を出す。最大表示時間1.5s、フェードアウト0.3s
2. **レベルアップ演出**: 画面中央に大きく表示（72px以上）+ スケールイン（0→1、0.2s、ease-out-back）+ 1秒後フェードアウト。背景に半透明暗幕（画面全体alpha=0.3、0.5s）
3. **アウトラインテキスト**: 全イベントテキストに2px黒アウトライン（LabelSettings shadow or custom drawing）
4. **MASSACRE等のコンボテキスト**: 右上固定位置。カウントアップアニメ。shake付き

### 2.8 XPジェム/ドロップ [2/5 → 目標3/5] — HIGH

**現状**:
- 小さなオレンジ/茶色のティアドロップ形状
- 背景と色が近く同化する（特にStage 2のオレンジ背景で致命的）
- グローなし、パルスアニメなし

**改善指示（builder）**:
1. **明るい色に変更**: XPジェムを鮮やかな青/紫/緑系に。背景色と補色関係にする
2. **小さなグロー追加**: ジェムの周囲に半径8pxのソフトグロー（PointLight2Dまたはadditive blend sprite）
3. **パルスアニメ**: scale を1.0→1.15→1.0でsin波（frequency=1.5Hz）。常時
4. **吸引エフェクト**: プレイヤーに引き寄せられる時に加速アニメ + 回収時にポップ音

### 2.9 画面密度 [2/5 → 目標3/5] — MEDIUM

**現状**:
- ゲーム序盤: 画面にプレイヤー1体 + 敵5-10体。**90%以上が空白**
- 中盤: 敵20-30体。まだ空白が多い
- SURROUNDED状態でようやく密度が上がるが、全てが赤いダイヤモンドなので単調

**改善指示（builder）**:
1. **地面タイルで空白を埋める**: 2.1参照。これだけで密度感が大幅改善
2. **環境オブジェクト散布**: 岩、木、壊れた柱等をフィールドに50-100個ランダム配置。衝突判定なし（純粋に装飾）。Kenney roguelike packの環境タイルを使用
3. **アンビエントパーティクル**: 2.1参照
4. **敵出現頻度**: ゲーム開始直後から画面内に最低15体の敵が見えるようにする（初期スポーン数調整）

### 2.10 カラーパレット [2/5 → 目標3/5] — MEDIUM

**現状**:
- 各ステージが1色の支配的色（紫、オレンジ、緑）
- ステージ内の色バリエーションが皆無
- プレイヤー/敵/背景のコントラスト不足

**改善指示（designer → builder実装）**:
1. **3色ルール**: 各ステージで「背景色」「プレイヤー陣営色」「敵陣営色」を明確に分離
   - Stage 1: 暗い青紫背景 / プレイヤー=明るい青白 / 敵=暖色（赤橙）✅既に近い
   - Stage 2: 暗い赤茶背景 / プレイヤー=明るい青白 / 敵=明るい黄緑
   - Stage 3: 暗い緑背景 / プレイヤー=明るい青白 / 敵=明るい赤紫
2. **プレイヤーは常に最も明るい**: プレイヤーキャラに明るいアウトラインor常時ソフトグローを追加
3. **XPジェムは補色**: 背景の補色でXPジェムを着色。常に目立つ

## 4. Brotato参考: なぜBrotatoが最適な目標か

| 要因 | 理由 |
|------|------|
| **Godot製** | 同じエンジン。技術的に完全再現可能 |
| **2Dアセット** | ピクセルアート不要。フラットな2Dスプライトで成立 |
| **CC0素材活用** | Kenney素材で十分なクオリティを達成している先例多数 |
| **上見すぎない** | VS/HoTの模倣は現実的でない。BrotatoレベルがHTML5の現実的目標 |
| **商業的成功** | Steam 98% Positive、300万+販売。品質証明済み |
